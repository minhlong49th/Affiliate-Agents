# Reference 05 — RSA Copywriting Guide
## PSBCU (Joey Babineau) × 17-Step Formula (John Crestani) — Combined

---

## Character Limit Rules (HARD — never violate)

| Asset | Limit | Notes |
|---|---|---|
| Headline | 30 chars | Includes spaces and punctuation |
| Description | 90 chars | Includes spaces and punctuation |
| Display Path 1 | 15 chars | Brand slug, no spaces |
| Display Path 2 | 15 chars | Intent word, no spaces |

**Always count characters before finalizing. When in doubt, shorten.**

## MANDATORY OUTPUT FORMAT FOR EVERY HEADLINE

Before outputting any headline, count characters (including spaces) and format as:
```
H[N] [PIN P1/P2 / —]: "[exact text]" | chars=[N] | [PASS if <=30 / FAIL — rewrite]
```
**FAIL rule**: If chars > 30 → rewrite immediately. Do NOT output a failing headline and flag later.
Auto-shorten method: remove lowest-value modifier → abbreviate → split to shorter phrase.
Max 3 rewrite attempts → if still > 30 chars after 3 attempts: `[MANUAL REVIEW REQUIRED]`

## PSBCU Framework Mapping to RSA Slots

| PSBCU Element | RSA slot(s) | AG1 Coupon weight | AG2 Review weight |
|---|---|---|---|
| **P — Problem** | H2–H3, D1 opening | Low (0–1 headlines) | Medium (2–3 headlines) |
| **S — Solution** | H3–H5 | "Coupon available here" framing | "Honest review / test results" framing |
| **B — Benefit** | H6–H9, D2 | Numbers: "Save X%" | Outcomes: "Proven in 30 days" |

| **U — Urgency** | H13–H15, D4 | Strong: "Ends Soon / Limited" | Soft: "Price may change" |

## 17-Step Formula Mapping to RSA Slots (Crestani — use for AG3/AG4 or description enrichment)

| Step | RSA element | Truncated for 30-char headline | Full in 90-char description |
|---|---|---|---|
| 1 Grab Attention | H1 (PINNED) | Brand + intent KW | N/A |
| 2 Identify Problem | H2–H3 | Question format | D1 opening sentence |
| 3 Agitate Problem | D1 | Short phrase | Full sentence in D1 |
| 4 Present Solution | H4–H5 | "Review Inside" / "Deal Found" | D1 closing |
| 5 Build Credibility | H6–H7 | "X+ Happy Customers" | D2 opening |
| 6 List Benefits | H8–H9 | Specific outcome | D2 body |
| 7 Social Proof | H10 | "Top Rated" / "Award Winning" | D2 closing |
| 8 Make the Offer | H11, D2 | "Up to X% Off" | D2 full offer detail |
| 9 Create Urgency | H12–H13 | "Limited Time" | D3 or D4 |
| 10 State the Price | H14 | "From $XX" | D3 |
| 11 Reduce Risk | H15 | "No Signup Needed" | D4 opening |
| 12 Restate Benefits | D3 | N/A | Full D3 |
| 13 Call to Action | D3 closing | N/A | "Click to reveal..." |
| 14 Warning/Scarcity | D4 | N/A | "Code may expire..." |
| 15–17 Final loop | D4 closing | N/A | Objection handling |

## Generation Instructions by Ad Group Type

### AG1 — Coupon Intent (PSBCU-dominant)

**Pin strategy:**
- ALWAYS pin H1 = `[Brand] Coupon Code [Year]` OR `[Brand] Promo Code [Year]`
- ALWAYS pin H2 = `Save [X%] Off Today` or `[X%] Discount Available`
- H3–H15 = unpinned

**Headline generation guide (generate all 15):**
```
H1 [PIN P1]: [Brand] Coupon Code [Year]         ← exact match to keyword
H2 [PIN P2]: Save Up to [X]% Off Today          ← specific benefit number
H3:          Verified [Brand] Promo Code         ← solution confirmation
H4:          Reveal Your Discount Now            ← CTA action
H5:          [Brand] Deals Today                 ← Safe default. "Official" is banned for affiliates on both platforms.
H6:          Free Shipping + [X]% Off            ← stacked benefit if applicable
H7:          Limited Time Offer                  ← urgency
H8:          [Brand] Deals – Updated Daily       ← credibility/freshness
H9:          [X]000+ Orders Placed               ← social proof (use if data found)
H10:         Unlock Your Savings Today           ← CTA variant
H11:         [Brand] [Product] on Sale           ← product-specific
H12:         Code May Expire Anytime             ← urgency variant
H13:         Top Rated + Best Price Online       ← benefit stacking
H14:         No Signup – Instant Code            ← risk removal (if applies to LP mechanic)
H15:         Shop [Brand] & Save Today           ← brand + benefit close
```

**Description generation guide (generate all 4):**
```
D1 (Problem + Solution):
"Stop paying full price for [Brand] products. We track all active coupon codes 
so you get the best price every time. Updated daily."
[Count chars — target 80–88 of 90]

D2 (Benefit + Offer + CTA):
"Click below to reveal your exclusive [Brand] promo code and save [X]% on 
[top product]. No email required – instant discount access."
[Count chars]

D3 (Social proof + Detail):
"Verified by [X] shoppers this month. [Brand] coupon works on all product 
lines including [product name]. Use at checkout."
[Count chars]

D4 (Objection + Final CTA):
"Worried the code won't work? We test every coupon before publishing. 
Click to reveal your working [Brand] discount code now."
[Count chars]
```

### AG2 — Product/Review Intent (PSBCU + 17-step hybrid)

**Pin strategy:**
- ALWAYS pin H1 = `[Brand] [Product Name] Review` OR `[Brand] Review [Year]`

**Pin strategy:** Pin H1 only — problem-hook question format.

**Headline generation guide (generate all 15):**

H2:           Why [Problem] Keeps Happening     ← Agitate: identify root cause

H9:           Top-Rated [Category] Solution     ← Social proof / award
H10:          Get [Solution] – No Risk          ← CTA + risk removal

H12:          Limited Supply Available          ← Urgency (if applicable)

H14:          From $[XX] – Best Price Found     ← Price anchor
H15:          Read Our [X]-Day Test Results     ← Bridge to LP content

**Description guide:**
D1: Agitate problem + introduce solution category (Crestani steps 2–4)
D2: Credibility + specific benefit + social proof (Crestani steps 5–7)
D3: Offer detail + CTA (Crestani steps 8–10)
D4: Risk removal + urgency + final CTA (Crestani steps 11–13)

## Self-check Before Finalizing Each Asset

For every headline, verify:
- [ ] ≤ 30 chars (count precisely including spaces)
- [ ] No exclamation mark
- [ ] No ALL CAPS (except acronyms, coupon codes)
- [ ] No emoji
- [ ] No "Official Site" claim
- [ ] No phone number
- [ ] Makes sense if shown alone (without other headlines)
- [ ] Different enough from other headlines (no near-duplicates)
- [ ] ≥ 3 words (Bing requirement)

For every description, verify:
- [ ] ≤ 90 chars
- [ ] Max 1 exclamation mark (prefer 0)
- [ ] No "Click Here" (especially Bing)
- [ ] Matches LP promise (no bait-and-switch)
- [ ] Contains CTA
- [ ] ≥ 3 words
